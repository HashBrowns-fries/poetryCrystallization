# Changelog 2026-06-04（续）— §4 label leakage 三档对照实验

## 摘要

完成审稿人第二条核心质疑的回应：**§4 BERT 体裁分类 F1≈0.98 是否来自 label leakage**。结论是 §4 站得住——体裁信号主要来自正文的词汇—语义内容，**保留为"语义证据"**，但报告口径从原始的 0.98 改为格式受控后的稳健数字（T1 三分类 0.919、T2 朝代受控 0.960）。

实验中发现了一条原 expC 从未控制、且比词牌名泄漏更隐蔽的**正字法泄漏通道**：本语料中诗（全唐诗/宋诗）以**繁体**存储，词曲以**简体**存储，CCPoem 简体词表对繁体文本产生约 17% 的 [UNK] 率、对简体仅约 0.2%，模型可据此"作弊"。

---

## 一、实验设计（`scripts/expM_label_leakage.py`）

同一模型（BERT-CCPoem mean-pool + CE）、同一超参、同一诗人级划分，**仅改变输入文本**：

**三档清洗：**
- **A 档**：词牌/曲牌名 + 标题 + 作者 + 正文，保留原始繁/简正字法与标点
- **B 档**：仅正文，去除全部元数据，保留原始正字法与标点
- **C 档**：仅正文，繁→简统一（OpenCC t2s）+ 去除全部标点/换行（仅保留汉字）

**两个任务（分离体裁与朝代共线）：**
- **T1**：shi/ci/qu 三分类（体裁↔朝代高度共线，随机基线 0.333）
- **T2**：宋诗 vs 宋词二分类（**朝代受控**，同为宋代仅体裁不同，随机基线 0.500）
  - shi 取自全唐诗库中的宋人诗（253,697 首），ci 取自宋词（20,938 首）

---

## 二、结果

| 任务 | 档 | Accuracy | Macro-F1 | [UNK]率 |
|------|----|---------:|---------:|--------:|
| T1（共线，基线0.333） | A 元数据+正文 | 0.966 | 0.966 | 0.167 |
|  | B 仅正文 | 0.972 | 0.972 | 0.169 |
|  | **C 归一化正文** | **0.919** | **0.919** | 0.002 |
| T2（朝代受控，基线0.500） | A 元数据+正文 | 0.999 | 0.999 | 0.199 |
|  | B 仅正文 | 0.999 | 0.999 | 0.198 |
|  | **C 归一化正文** | **0.960** | **0.960** | 0.003 |

**三点结论：**

1. **元数据泄漏被排除**：A→B 去除标题/词牌名/作者后精度未降（T1 反升 0.966→0.972）。审稿人担心的词牌名/标题污染**不成立**。

2. **格式泄漏量级有限**：B→C 统一正字法并去标点后 [UNK] 率 0.17→0.002，证实正字法是真实泄漏通道；但精度仅降约 5 个百分点（T1 0.972→0.919；T2 0.999→0.960），远未坍塌。

3. **体裁信号在朝代受控且格式归一后依然稳健**：最严格的 T2-C 档（同朝代、无元数据、无正字法/标点线索）仍达 0.960（基线 0.500，绝对提升 0.46）。
